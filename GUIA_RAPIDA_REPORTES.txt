 1. 🔐 ACCESO AL SISTEMA
═══════════════════════════════════════════════════════════════════════════════

URL: http://localhost/mayra/src/public/admin-reportes

REQUISITOS:
  ✓ Usuario: Administrador o Empleado

═══════════════════════════════════════════════════════════════════════════════
 📌 TIPS Y MEJORES PRÁCTICAS
═══════════════════════════════════════════════════════════════════════════════

✓ FRECUENCIA RECOMENDADA:
  • Reporte General: Diario (cada mañana)
  • Reporte Ventas: Semanal y Mensual
  • Reporte Productos: Semanal
  • Reporte Clientes: Mensual
  • Reporte Inventario: Diario o Semanal

✓ EXPORTACIÓN:
  • Excel: Para análisis numérico y gráficos personalizados
  • PDF: Para presentaciones y archivo formal
  • Imprimir: Para reuniones físicas

✓ INTERPRETACIÓN:
  • Gráficos ascendentes = Crecimiento (bueno)
  • Gráficos descendentes = Decrecimiento (revisar)
  • Picos en ventas = Identificar causa (promociones, eventos)
  • Stocks bajos = Acción inmediata requerida

✓ COMPARACIONES:
  • Comparar mes actual vs mes anterior
  • Comparar semana actual vs semana anterior
  • Buscar patrones (días de más ventas, etc.)

✓ ACCIONES:
  • Stock bajo → Hacer pedido a proveedor
  • Top clientes → Programa de fidelización
  • Productos sin ventas → Promoción o descontinuar
  • Ventas bajas → Revisar estrategia de marketing


═══════════════════════════════════════════════════════════════════════════════
 🆘 SOLUCIÓN DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: No se generan datos
┌────────────────────────────────────────────────────────────────────────────┐
│ POSIBLES CAUSAS:                                                           │
│  • No hay pedidos en el período seleccionado                              │
│  • Fechas incorrectas (fecha fin antes de fecha inicio)                   │
│  • Base de datos sin conexión                                             │
│                                                                            │
│ SOLUCIÓN:                                                                  │
│  1. Verificar que haya pedidos en el período                              │
│  2. Cambiar el período a "Este Mes"                                       │
│  3. Verificar que MySQL esté activo en XAMPP                              │
└────────────────────────────────────────────────────────────────────────────┘

PROBLEMA: Exportación no funciona
┌────────────────────────────────────────────────────────────────────────────┐
│ POSIBLES CAUSAS:                                                           │
│  • No se generó el reporte antes de exportar                              │
│  • Bloqueador de pop-ups activo                                           │
│  • Permisos de descarga bloqueados                                        │
│                                                                            │
│ SOLUCIÓN:                                                                  │
│  1. Generar reporte primero, luego exportar                               │
│  2. Permitir descargas en el navegador                                    │
│  3. Desactivar bloqueador de pop-ups para este sitio                      │
└────────────────────────────────────────────────────────────────────────────┘

PROBLEMA: Gráficos no se muestran
┌────────────────────────────────────────────────────────────────────────────┐
│ POSIBLES CAUSAS:                                                           │
│  • Chart.js no cargó correctamente                                        │
│  • Conexión a internet inestable (CDN de Chart.js)                        │
│  • JavaScript deshabilitado                                               │
│                                                                            │
│ SOLUCIÓN:                                                                  │
│  1. Refrescar la página (F5 o Ctrl+F5)                                    │
│  2. Verificar conexión a internet                                         │
│  3. Habilitar JavaScript en el navegador                                  │
│  4. Revisar consola del navegador (F12) para errores                      │
└────────────────────────────────────────────────────────────────────────────┘
